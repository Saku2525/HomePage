Gitメモ

Git用語

・記録を保存する操作を「コミットする」。また、その記録自体を「コミット」という。

・　　　リポジトリ　　　　→　コミットをためていく場所。

・　ローカルリポジトリ　　→　自分専用のリポジトリ。

・　リモートリポジトリ　　→　ネットワーク上のリポジトリ。

・　　　　クローン　　　　→　リポジトリをコピーすること。

・ディレクトリ（CUI）

　―ルートディレクトリ　　→　一番階層が浅いディレクトリ。

　―ホームディレクトリ　　→　初期表示されるディレクトリ。

　―カレントディレクトリ　→　現在のディレクトリ。

・　　　　　差分　　　　　→　ファイルの修正前と修正後の違い。

・　　　　フォーク　　　　→　リポジトリの複製。

・　　　　origin　　　　　→　リモートリポジトリ。

・　　　　ブランチ　　　　→同じリポジトリ内で並行して作業する際に必要な機能。

・　　　　マージ　　　　　→ブランチを融合すること。

Gitコマンド

pwd カレントディレクトリのパスの確認。
mkdir
相対パス/絶対パス		 新しいディレクトリの作成。
ls ディレクトリ内のディレクトリ・ファイルの確認。
ls -a すべてのディレクトリ・ファイルの表示。
cd
相対パス/絶対パス		 カレントディレクトリの移動。
git config --list 設定内容の確認。
git init カレントディレクトリにローカルリポジトリの作成。
git clone SSH クローンの実行。
git status ローカルリポジトリの状態の確認。
git add
相対パス/絶対パス		 ファイルをステージングエリアに登録。
git diff ワークツリーとステージングエリアの差分の確認。ステージングエリア登録後にどんな変更をしたかが分かる。
git diff --cached ステージングエリアとGitディレクトリの差分の確認。ステージングエリアにどんな変更が登録されているかが分かる。
git commit コミットする。
git commit -m
"コミットメッセージ"		 直接コミットメッセージを指定。
git log コミットの履歴の確認。
git log -p 差分付きのコミットの履歴の確認。
git remote リモートリポジトリの設定を確認・変更。
git remote -v URL情報の確認
code 相対パス/絶対パス エディターの表示。
git branch
作成するブランチ名		 ブランチの作成。
git checkout
切り替え先のブランチ名		 ブランチを切り替え、作業する。
